Preface:

Please note: Because Gisto is in early stage of its life - many documented aspects are subject to change and should not be considered final nor absolute.

Tag gist

Every gist can be easy tagged by adding hash-tag to it's title.
For example: You have a gist titled

{% highlight text %}Simple html template{% endhighlight %}

and you'd like to tag it with "template" and "html" tags. All you need to do, is to add this tags as hash-tags to the end of the title like so:

{% highlight text %}Simple html template #html #template{% endhighlight %}

Gisto will then use the hash-tags to display tags in the gist list and search by this tags.

Setting up for development

Pre-installed requirements:

- IDE of your choosing

- GIT - distributed version control

- Node.js - server-side software system written in JavaScript

- Node-webkit - app runtime based on Chromium and node.js

- Grunt - the JavaScript Task Runner

Set it all to work together:

- Clone the latest source with the following command to a directory of your choice:

{% highlight bash %}git clone https://github.com/Gisto/Gisto.git{% endhighlight %}

- Enter into the newly created directory:

{% highlight bash %}cd Gisto{% endhighlight %}

- Run:

{% highlight bash %}npm install{% endhighlight %}

this will install all necessary dependencies.

- Run:

{% highlight bash %}grunt{% endhighlight %}

- You can download a 'web-server.js' script used in AngularJS phonecat tutorial to gisto directory.

- Run:

{% highlight bash %}node web-server.js{% endhighlight %}

- (optionaly) Run:

{% highlight bash %}grunt watch{% endhighlight %}

- Navigate your Chrome/Chromium browser to http://localhost:8000/app/index.html

Congrats! If all went well, you are now running local version of Gisto in your machine.
